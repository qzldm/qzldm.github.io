如何在hexo发布文章并编辑文章

发表于2023-04-17|更新于2023-04-20|博客建设
|阅读量:

Hello world

这是我的第一篇博客，是在vscode中编写的，使用的语法是markdown进行编写，我将展示如何创建文章及书写文章。

git bash创建文章

在网站根目录下git bash here，输入hexo new "title"即可创立
创立文章所在处：D:\Blog\source_posts

文章属性

文章属性学习，主要是关于标题，日期，封面等等的配置

Markdown语法编辑文章

首先要注意自己所用的markdown编辑器的渲染版本是否与hexo的一致，如果不一致会出现不同的渲染结果，也即会导致界面不同。
vscode中markdown preview enhanced所使用的markdown渲染器为hexo-renderer-markdown-it-plus。
注意每一个符号后都要跟随空格

标题

#为一级标题
##为二级标题
以此类推

段落格式

Markdown段落没有特殊的格式，直接编写文字就行，段落的换行是使用两个以上空格加上回车
你好
也可以在段落后面空一行表示重新开始一个段落

字体

可以用和_实现，两者等价，但不可混用
斜体字体（单星号包围， 字段 *，*后不可空格）
斜体字体（单_）
粗体字体（双星号）
粗体字体(双_)
粗斜体字体(三星号)
粗斜体字体（三_）

分割线

三个*或_或-及以上建立一个分割线

删除线

不是
~~ 不是 ~~
双~~包围,~~后不可空格，且不可用符号结尾

下划线

带下划线文本
可通过HTML的<u>标签开始，</u>结束

脚注

这是一个脚注[1]。

[^引用ID]:说明文字

列表

  • 第一项（用*）
  • 第二项（第一级）
    • 你好（第二级）
      • 我好（第三级）
  • 第一项（用+）
  • 第二项
  • 第一项（用-）
  • 第二项

无序列表使用*，+，-作为列表标记。
列表嵌套只需在子列表中的选项前添加两个或四个空格即可（按一次tab）

  1. 有序列表
  2. 有序列表
    • 子列表
    • 子列表

有序列表只需在数字后加 . 即可

区块

区块引用是在段落开头使用>符号，后面紧跟一个空格符号

区块引用举例

区块嵌套

1.区块列表
2.区块列表

  • 第一项

    列表区块
    列表区块

以下是对应上例的书写的格式

> > 区块嵌套     
>
> 1.区块列表
> 2.区块列表
* 第一项
    > 列表区块
    > 列表区块

代码

  • 如果是段落上的一个函数或片段可以用反引号(`)把它包起来
  • 代码区块使用```包括一段代码，可指定一种语言(非必要)
cout<<"hello world"<<endl;

链接

[链接名称](链接地址)或<链接地址>
欢迎访问我的github
图片地址则前面需加!，链接则没有

图片插入

![属性文本](图片地址)
注意图片地址必须是网络地址加上?raw=true(如果是webp则不需要加),预览可以出现图片但是博客中出现不了
解决方案

另一种方式：使用HTML终端<img>标签,可以指定图片高度和宽度,img元素教程
图片地址举例：
https://github.com/qzldm/qzldm.github.io/blob/main/img/K-on.jpg?raw=true

（最推荐）最终解决方案，使用图床保存自己的图片，再复制图片链接，使用html标签显示

表格

表格制作

Markdown指标格使用|来分隔不同的单元格，使用-来分隔头和其他行

表头 表头
单元格 单元格 
| 表头 | 表头 |
| ---- | ---- |
| 单元格|单元格|

对齐方式

单元格默认左对齐标题栏(当表头长时)
表头默认居中对齐单元格(当单元格长时)
-:设置内容与标题栏右对齐
:-左对齐
:-:居中

其他技巧

支持HTML元素

Markdown 支持HTML元素，不在Markdown涵盖范围内的标签都可以直接在文档里面用HTML撰写

转义

Markdown使用反斜杠转义特殊字符\

公式

Markdown Preview Enhanced可以使用KaTeX或MathJax来渲染数学表达式
graph LR

流程图

1. 横向流程图

mermaid//这是代码的语言名称

    B --> C{条件a}
    C -->|a=1| D[结果1]
    C -->|a=2| E[结果2]
    F[横向流程图]

2. 竖向流程图

mermaid
graph TD
A[方形] --> B(圆角)
    B --> C{条件a}
    C --> |a=1| D[结果1]
    C --> |a=2| E[结果2]
    F[竖向流程图]

3. 标准流程图源码格式

flow
st=>start: 开始框
op=>operation: 处理框
cond=>condition: 判断框(是或否?)
sub1=>subroutine: 子流程
io=>inputoutput: 输入输出框
e=>end: 结束框
st->op->cond
cond(yes)->io->e
cond(no)->sub1(right)->op

4. 标准流程图源码格式（横向）

flow
st=>start: 开始框
op=>operation: 处理框
cond=>condition: 判断框(是或否?)
sub1=>subroutine: 子流程
io=>inputoutput: 输入输出框
e=>end: 结束框
st(right)->op(right)->cond
cond(yes)->io(bottom)->e
cond(no)->sub1(right)->op

5. UML时序图源码样例

sequence
对象A->对象B: 对象B你好吗?（请求）
Note right of 对象B: 对象B的描述
Note left of 对象A: 对象A的描述(提示)
对象B-->对象A: 我很好(响应)
对象A->对象B: 你真的好吗？
